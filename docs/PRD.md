## 1. 專案概述

### 1.1 背景
LINE 是台灣最普及的即時通訊平台，擁有龐大的活躍用戶群。與此同時，個人投資理財風氣日盛，散戶投資人有隨時查詢股價的需求。Gemini 2.5 Flash 是 Google 最新的高效能 LLM，具備優秀的中文對話能力與低延遲特性，非常適合作為 Chatbot 的推理引擎。

### 1.2 問題陳述
目前股票散戶投資人若想查詢即時股價，需要切換至不同 App（如證券 App、Yahoo 股市、Google Finance），無法在 LINE 日常通訊情境中直接取得資訊。此外，現有股票查詢工具缺乏自然語言對話能力，使用者只能被動查詢數字，無法進行深入問答或投資諮詢。

### 1.3 解決方案摘要
開發一個部署於 LINE Messaging API 的智慧 Bot，整合 Gemini 2.5 Flash 作為對話大腦，並串接即時股價數據 API。使用者只需在 LINE 中以自然語言輸入需求（例如「台積電現在多少？」），Bot 即可自動判斷意圖，返回即時股價或進行 AI 對話，讓投資資訊唾手可得。

## 2. 目標與成功指標

### 2.1 業務目標
- 建立一個可運作的 MVP LINE Bot，展示 AI + 金融資訊整合的可行性
- 降低使用者查詢股票資訊的操作步驟，從切換 App（3 步以上）縮短至 LINE 內直接提問（1 步）
- 為後續擴充更多金融功能（如技術分析、投資組合管理）建立基礎架構

### 2.2 使用者目標
- 在 LINE 對話視窗中，用一句自然語言即可取得指定股票的即時股價
- 可透過 AI 對話詢問股票相關問題，獲得有意義的回應
- 不需要學習特殊指令語法，降低使用門檻

### 2.3 成功指標（KPI）

| 指標 | 目標值 | 衡量方式 |
|------|--------|---------|
| 股價查詢回應時間 | < 3 秒 | 從使用者送出訊息至 Bot 回覆的時間 |
| AI 對話回應時間 | < 5 秒 | 從使用者送出訊息至 Bot 回覆的時間 |
| Webhook 回應時間 | < 1 秒 | LINE 平台要求（避免逾時重送） |
| 股價查詢成功率 | ≥ 95% | 成功返回股價筆數 / 總查詢筆數 |
| Bot 系統可用性 | ≥ 99% | 每月監控數據 |

## 3. 目標使用者

### 3.1 主要使用者畫像

**角色名稱**：台股散戶小明  
**描述**：25–45 歲，有工作、兼顧投資的上班族或學生，每天使用 LINE 頻率極高，持有台股或美股部位，習慣在手機上快速瀏覽資訊  
**痛點**：  
- 想在通勤或開會空檔快速確認股價，但切換 App 太麻煩  
- 看到財經新聞有問題想問，但沒有方便的管道快速得到解答  
- 不想學習複雜的指令格式  

**需求**：  
- 能在 LINE 中直接用中文問「XXX 股票現在幾塊？」
- 能問一些基本的股票相關問題並獲得 AI 解說  

### 3.2 使用場景

- **場景 1**：小明在捷運上看到台積電的新聞，直接在 LINE 傳訊息給 Bot「台積電現在多少？」，Bot 立刻回覆即時股價與漲跌幅
- **場景 2**：小明不確定 P/E ratio 是什麼，直接問 Bot「本益比是什麼意思？」，Bot 用 Gemini 生成清楚易懂的解說
- **場景 3**：小明想同時查多支股票，分別傳訊息查詢 TSMC、NVDA，Bot 逐一正確回應

## 4. 功能需求

### 4.1 必要功能（P0 - Must Have）
> 這些功能不完成則無法上線

| 功能 ID | 功能名稱 | 描述 | 驗收標準 |
|---------|---------|------|---------|
| F-001 | LINE Webhook 接收訊息 | 接收 LINE 平台傳來的使用者文字訊息事件，通過簽章驗證後進行處理 | 發送訊息後 Bot 在 1 秒內回應（不含 AI 處理時間）；LINE 平台不觸發重送機制 |
| F-002 | Gemini 2.5 Flash AI 對話 | 將使用者輸入送至 Gemini 2.5 Flash API，取得回覆後透過 LINE Reply API 回傳給使用者 | 任意輸入中英文訊息，Bot 能在 5 秒內回覆語意相關的 AI 回應；API 錯誤時回傳友善錯誤訊息 |
| F-003 | 即時股價查詢（意圖識別） | 系統能識別使用者訊息中包含的股票查詢意圖（含股票代號或公司名稱），自動觸發股價查詢流程 | 輸入「台積電現在多少」、「2330 股價」、「查 AAPL」等語句，Bot 能正確識別為股價查詢並回傳股價資料 |
| F-004 | 即時股價取得與回傳 | 根據識別出的股票代號，呼叫股價資料 API 取得即時（或延遲 15 分鐘）股價，格式化後回傳 | 回傳資訊包含：股票名稱、股票代號、當前股價、漲跌金額、漲跌幅（%）、資料時間戳；回應時間 < 3 秒 |
| F-005 | 環境變數設定 | 所有 API 金鑰（LINE Channel Secret、Access Token、Gemini API Key、股價 API Key）透過環境變數管理，不寫入程式碼 | 服務能透過 `.env` 檔案正確讀取所有金鑰並啟動；任何金鑰缺失時，程式啟動即報錯並提示缺少的變數名稱 |

### 4.2 重要功能（P1 - Should Have）
> 重要但可延後至下一個迭代

| 功能 ID | 功能名稱 | 描述 | 驗收標準 |
|---------|---------|------|---------|
| F-101 | 多輪對話記憶 | 在同一使用者的對話 Session 中，保留近 N 輪對話歷史，使 Gemini 能根據上下文回覆 | 連續對話時，Gemini 能正確引用前一輪內容；Session 在 30 分鐘無互動後自動清除 |

## 6. UI/UX 設計

### 6.1 設計原則
- **直覺操作**：使用者無需閱讀說明即可使用，自然語言即為操作介面
- **快速回饋**：回應內容簡潔，資訊結構清晰，股價資訊一目了然
- **友善的錯誤引導**：失敗時提供明確的替代建議，避免使用者卡住

### 6.2 使用者流程

**股價查詢流程：**
```
[使用者輸入含股票關鍵字的訊息]
    → [Webhook 接收事件]
    → [意圖識別：判斷為股價查詢]
    → [擷取股票代號/名稱]
    → [呼叫股價 API]
    → [格式化回覆訊息]
    → [LINE Reply API 回傳給使用者]
```

**AI 對話流程：**
```
[使用者輸入一般問題訊息]
    → [Webhook 接收事件]
    → [意圖識別：判斷為一般對話]
    → [帶入對話歷史呼叫 Gemini 2.5 Flash API]
    → [取得 AI 回覆]
    → [LINE Reply API 回傳給使用者]
```

**股價回覆訊息格式範例：**
```
📈 台積電 (2330)
當前股價：$980.00
漲跌：▲ +12.00 (+1.24%)
更新時間：2026-04-29 09:30
```

## 7. 技術考量

### 7.1 系統架構概述

採用後端服務架構（無前端），部署一個 Python Web Server（建議使用 FastAPI 或 Flask）作為 LINE Messaging API 的 Webhook 端點。伺服器接收訊息後進行意圖判斷，分流至 Gemini AI 對話模組或股價查詢模組，最終透過 LINE Reply API 回傳結果。

```
LINE 使用者
    ↕ LINE App
LINE Platform (Messaging API)
    ↕ HTTPS Webhook
Python Web Server (FastAPI/Flask)
    ├── 意圖識別模組
    ├── Gemini 2.5 Flash 對話模組 ←→ Google Gemini API
    └── 股價查詢模組 ←→ 股價資料 API (yfinance / Alpha Vantage / Fugle)
```

### 7.2 第三方依賴

| 服務/工具 | 用途 | 備註 |
|----------|------|------|
| LINE Messaging API | 接收使用者訊息、發送回覆 | 需建立 LINE Developers 帳號，設定 Channel |
| Google Gemini 2.5 Flash API | AI 自然語言對話 | 需申請 API Key，注意用量計費 |
| 股價資料 API（yfinance / Alpha Vantage / Fugle） | 取得即時或延遲股價資料 | yfinance 免費但依賴 Yahoo Finance；Fugle 支援台股即時報價（需申請）；Alpha Vantage 免費方案有速率限制 |
| Python line-bot-sdk | LINE Webhook 處理與回覆發送 | 官方 SDK，版本需對應 Messaging API v3 |
| google-generativeai | 呼叫 Gemini API | Google 官方 Python SDK |
| python-dotenv | 環境變數管理 | 從 .env 檔案讀取設定 |
| ngrok（開發階段） | 本地開發時暴露 Webhook 端點 | 僅開發用，正式部署需雲端伺服器 |

### 7.3 資料需求
- **需要儲存的資料**：對話歷史（P1 功能，每位使用者近 10 輪，存於記憶體或 Redis）；股票代號與公司名稱對照表（靜態資料或 CSV 檔案）
- **資料保留政策**：對話 Session 30 分鐘無互動後自動清除；不持久化儲存使用者訊息內容（隱私保護）

## 8. 限制與假設

### 8.1 已知限制
- **LINE 平台 Webhook 逾時**：LINE 要求 Webhook 在 1 秒內回應 HTTP 200，若 Gemini API 或股價 API 耗時較長，需採用非同步處理（先回應 200，再非同步發送 Push Message）
- **股價資料延遲**：使用免費 API（如 yfinance）時，台股股價可能有 15–20 分鐘延遲，需在回覆中標明資料時間
- **Gemini API 費用**：用量超出免費額度後需付費，需監控 API 用量
- **意圖識別準確性**：MVP 階段以關鍵字規則比對為主，可能有誤判情況

### 8.2 假設前提
- 假設使用者已加入 LINE Bot 好友
- 假設股票查詢以台股為主（股票代號為 4–6 位數字），美股支援列為 P2
- 假設 MVP 階段以一對一私訊為主，群組支援列為 P2
- 假設部署環境為雲端 VPS（如 Render、Railway、GCP Cloud Run），需能對外提供 HTTPS 端點
